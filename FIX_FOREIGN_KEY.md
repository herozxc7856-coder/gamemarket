# 🚨 Ошибка Foreign Key — Быстрое решение

## ❌ Проблема:
```
Foreign key constraint violated on the foreign key
Code: P2003
```

Это значит: **в базе данных нет игры или пользователя**, на которые ссылается товар.

---

## ✅ Решение за 30 секунд:

```cmd
# 1. Откройте Terminal ОТ ИМЕНИ АДМИНИСТРАТОРА
# 2. Убедитесь, что ВПН включён
# 3. В папке проекта выполните:

setup-db.bat
```

**ИЛИ вручную:**
```cmd
cd C:\Users\genius\Downloads\gamemarket
npx prisma generate
npx prisma db push --accept-data-loss
node scripts/setup-db.js
npm run dev
```

---

## 🎯 Что сделает setup-db.bat:

```
✅ Создаст игры в базе (WoT, CS2, GTA V...)
✅ Создаст тестового пользователя (ID: default-seller)
✅ Создаст тестовый товар для проверки
✅ Подготовит базу для работы
```

---

## 🔄 Если setup-db.bat тоже выдаёт ошибку:

### Вариант А: Проверьте базу вручную
```cmd
# Откройте папку проекта и проверьте:
dir prisma\dev.db
```
Если файла нет — база не создана. Попробуйте снова с включённым VPN.

### Вариант Б: Используйте mock-режим (без БД)
```cmd
# Интерфейс будет работать, данные не сохраняются:
npm run dev
```
> Товары создаются "виртуально" — для тестирования дизайна.

### Вариант В: Минимальная версия
```cmd
copy package.minimal.json package.json
npm install --legacy-peer-deps --timeout=300000
npm run dev
```
