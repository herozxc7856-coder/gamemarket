# 🎮 GameMarket (NexaTrade) — Инструкция по запуску

## ✅ Что уже реализовано

### 🖼️ Загрузка фото
- **API**: `/api/upload` — загрузка изображений (POST)
- **Поддержка**: JPEG, PNG, WebP, GIF до 10MB
- **Лимит**: до 10 фото на товар
- **Drag & Drop** + выбор файлов

### 💰 Система комиссий
- **Базовая ставка**: 10% (настраивается)
- **Категории**: разные ставки для аккаунтов, валюты, буста и т.д.
- **Скидки**: -2% для проверенных продавцов, -3% для топ-продавцов
- **Авто-расчёт**: в реальном времени при создании товара

### 🛒 Продажа товаров
- **Страница**: `/sell` — форма создания объявления
- **Шаги**: 1) Информация → 2) Цена/Фото → 3) Подтверждение
- **Превью**: отображение загруженных фото
- **Валидация**: проверка всех полей перед отправкой

## 🚀 Быстрый запуск (через npm с зеркалом)

### ⚠️ Важно: Запускайте терминал ОТ ИМЕНИ АДМИНИСТРАТОРА!

### Способ 1: Готовый скрипт (рекомендуется)
```cmd
cd C:\Users\genius\Downloads\gamemarket
install-ru.bat
```

### Способ 2: Вручную (если скрипт не работает)
```cmd
# 1. Откройте PowerShell ОТ ИМЕНИ АДМИНИСТРАТОРА (Win+X → Terminal Admin)

# 2. Перейдите в папку проекта:
cd C:\Users\genius\Downloads\gamemarket

# 3. Настройте российское зеркало:
npm config set registry https://registry.npmmirror.com

# 4. Очистите кэш:
npm cache clean --force

# 5. Удалите старые файлы:
rmdir /s /q node_modules 2>nul
del package-lock.json 2>nul

# 6. Установите зависимости (может занять 5-10 минут):
npm install --legacy-peer-deps --ignore-scripts --timeout=300000

# 7. Запустите сервер:
npm run dev

# 8. Откройте в браузере:
# 👉 http://localhost:3000
```

## 🆘 Если ничего не помогает — Минимальная версия

Если установка зависимостей постоянно падает с ошибками сети:
```cmd
# 1. Прочитайте инструкцию:
notepad README_MINIMAL.md

# 2. Или выполните быструю установку минимальной версии:
copy package.minimal.json package.json
npm config set registry https://registry.npmmirror.com
npm cache clean --force
rmdir /s /q node_modules 2>nul
npm install --legacy-peer-deps --timeout=300000
npm run dev
```

> ⚠️ Минимальная версия — **только фронтенд**. Интерфейс работает, но данные не сохраняются.
> Полную версию с БД можно настроить позже, когда сеть будет стабильнее.

## ⚙️ Настройка комиссий

Откройте `src/lib/commission.ts` и измените:

```typescript
export const COMMISSION_CONFIG = {
  defaultRate: 0.10,        // 10% по умолчанию
  minRate: 0.05,            // Минимум 5%
  maxRate: 0.25,            // Максимум 25%
  
  categoryRates: {
    'accounts': 0.15,       // Аккаунты
    'currency': 0.12,       // Валюта
    'boost': 0.10,          // Буст
    'items': 0.08,          // Предметы
    'services': 0.10,       // Услуги
  },
  
  verifiedSellerDiscount: 0.02,  // Скидка проверенным
  topSellerDiscount: 0.03,       // Скидка топ-продавцам
};
```

## 🔧 Устранение проблем

### ❌ Ошибка "Bun not found"
```cmd
# Установите Bun вручную:
# 1. Скачайте с https://bun.sh/install
# 2. Или через winget:
winget install Oven.Bun
```

### ❌ Ошибка Prisma
```cmd
# Переустановите клиент:
bunx prisma generate
bunx prisma db push
```

### ❌ Ошибка загрузки фото
- Проверьте, что папка `public/uploads/products` существует
- Убедитесь, что файлы < 10MB и в формате JPG/PNG/WebP/GIF

### ❌ Порт 3000 занят
```cmd
# Запустите на другом порту:
bun run dev -p 3001
# Откройте: http://localhost:3001
```

## 🎯 Как протестировать

1. **Запустите проект** (см. выше)
2. **Перейдите на** `http://localhost:3000/sell`
3. **Заполните форму**:
   - Выберите игру
   - Введите название и описание
   - Укажите цену → увидите расчёт комиссии
   - Загрузите 1-2 фото (перетащите или выберите)
4. **Нажмите "Опубликовать"**
5. **Проверьте**: товар появится в каталоге
